-----

zoomed-in plans


new code regions:
    mourningwail:
        new django app for metrics/analytics
    mourningwail.metrics.events:
        models for storing event-stream data in elasticsearch
    mourningwail.metrics.reports:
        models for storing report data in elasticsearch
    mourningwail.reporters: 
        given a date, a reporter generates data about what happened on that date
        `reporter.report(date)` returns iterable of `mourningwail.metrics._base.DailyReport` instances



new api surface:
    /_/mw/report/<report_name>/latest/
    /_/mw/report/<report_name>/recent/?days_back=7
    /_/mw/event/page_visit/ (POST new page-visit events)
    /_/mw/query/node_analytics/<guid>/<timespan>/ (GET everything needed for the node analytics page)


sometimes, the data we want is already in a daily report,
so let's allow getting reports within a date range


well-constrained query endpoint:
    start simple:
        COUNT
        SUM (for integer fields on reports)
    require a timespan
    allow bucketing by time interval
    decouple api field names from elasticsearch field names
